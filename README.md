## 🧭 How I Think About Building

```
Research → Frame the Problem → Architect → Prototype → Ship → Iterate
```

> Not every step is linear — but the thinking always starts with *why* before *how*.

| Phase | What happens |
|---|---|
| 🔍 **Research** | User interviews, competitive teardowns, data analysis |
| 🏗️ **Architecture** | System design with scalability, modularity, and trade-offs in mind |
| 💡 **Framing** | Turning ambiguity into a well-scoped problem statement |
| 🗺️ **Prioritization** | RICE, MoSCoW, impact–effort matrices — not gut feel |
| 🔁 **Iteration** | Tight feedback loops, not big-bang launches |
